# Pre-PR Self-Review Checklist

Before opening a PR, I check the following:

- [ ] Code does what the title says — no hidden surprises
- [ ] Tests exist for new features or bug fixes, and all tests pass
- [ ] README or other docs updated if anything changed
- [ ] Only one logical change per PR — no unrelated extras
- [ ] No debug code left behind (print statements, breakpoints, etc.)
- [ ] Code is readable and follows project style
- [ ] Dependencies are added only if necessary, with context
- [ ] “How to run” instructions work on a clean machine
